GitHub平台上Hexo博客系统的搭建

发表于 2018-11-23 | 分类于 Hexo ， GitHub | 评论数： | 热度：

一、关于GitHub + Git + Node.js + Hexo平台的相关信息

　　在搭建平台之前，我们有必要先来了解一下有关GitHub、Git、Node.js、Hexo等的相关信息，这样我们才能有的放矢。

1. 关于GitHub

  • GitHub 是一个面向开源及私有软件项目的托管平台，于2008年4月10日正式上线，2018年6月4日微软宣布：通过75亿美元的股票交易收购代码托管平台GitHub。GitHub对公共开源项目完全免费，可以为基于GitHub Pages服务托管的静态网页提供免费的服务器和空间，因此我们就可以很轻松的将我们的博客或网页部署托管在GitHub上。

  • GitHub Pages是独立的部署在GitHub的空间和服务器上，不会像国内大多数的博客网站一样屏蔽一大堆的敏感词汇或强行删除你的博客。但GitHub建在国外，链接速度会有一定延迟或不稳定，对百度的搜索引擎支持不是很友好，需要注意的是GitHub的免费项目都是公共的，也就是说你在GitHub上的所有文件大家都是可以看到的，如果你原本就是打算公开且未涉及个人隐私，那就完全没问题；但如果你对此无法接受或打算发布上传牵扯到个人隐私的文件的话，推荐你可以选择放弃或成为付费用户或选择提供免费私人项目托管平台的提供商。目前比较有名的托管平台提供商，大概就是国外的GitHub，BitBucket，GitLab以及国内的Coding(目前免费项目貌似和腾讯合作升级为腾讯云开发者平台)，具体操作大同小异，请自行了解。

2. 关于Git

  • Git 是一个开源的分布式版本控制系统，可以有效、高速的处理各种项目版本管理。GitHub上的项目就是依托Git进行版本库同步管理的，可以把我们的本地博客同步托管到GitHub上。

  • Git能够允许我们使用自己的个人电脑进行本地化的博客创建，创作并不需要在网络环境下，大大降低了创作的难度，使思绪更加天马行空，然后通过简单的命令随时同步自己的文章。

3. 关于Node.js

  • Node.js 是一个Javascript运行环境，能够让JavaScript运行在服务端的开发平台。

  • Node.js是基于 Chrome JavaScript 运行时建立的平台，用于方便地搭建响应速度快、易于扩展的网络应用。

4. 关于Hexo

  • Hexo 是基于Node.js的开源博客系统，通过Markdown编辑制作静态博客，利用自定义的主题自动生成靓丽的静态网页。

  • Hexo 可以使用 Markdown 语法进行文本编辑，使页面的排版更快捷美观。或者使用Markdown编辑程序，你所要做的就是进行文字的输入，并且能够实时预览页面样式。Hexo拥有众多的主题和扩展可供选择，你可以随时更换炫酷的主题来使自己的博客与众不同。

　　因此，如果把GitHub看作一台计算机的话，那么Git就好比是操作系统，Node.js则是程序运行的环境，Hexo就应该是其中运行的程序并制作出我们希望看到的作品。其实整个过程并不是像我们比喻的如此，我们使用基于Node.js的博客系统Hexo在我们的个人电脑上创建本地化的博客内容并生成静态网页文件，然后通过Git将我们的网页文件同步到GitHub的个人空间，这样我们就能通过GitHub的服务器和分配的网址在网络上浏览到的我们网页信息了。

二、搭建平台

　　下面我们将进行平台的搭建，我本人没有相关方面的知识，也是通过网络查询而来，发现很多由于组件版本更新导致的变化，特此记录于此，以便以后自己查看和为有同样需求的网友提供参考。

1. 安装 Git

  • 请到Git官网下载，一些朋友如果无法下载，可以前往GitHub上热心人士提供的国内下载地址[点击跳转]。本人版本为V2.19.1.1

  • 下载完成后点击安装，除了修改安装目录外，其他选项全部默认，一路 next 到底后等待安装完成即可。这时回到桌面点击右键，理论上应该会有“Git Bash Here”和“Git GUI Here”两个快捷键

2. 安装 Node.js

  • 请到Node.js 官网下载，本人版本为v10.13.0

  • 下载完成后点击安装，除了修改安装目录外，其他选项全部默认，一路 next 到底后等待安装完成即可。

命令 命令简写 描述

hexo new page “标题” hexo n page “标题” 新建页面
hexo server hexo s 本地预览,Ctrl+C关闭
hexo generate hexo g 生成本地静态网页
hexo deploy hexo d 部署网页到仓库
hexo clean 清除缓存
cd 切换操作路径,如 cd /d/hexo 就是切换到D盘hexo文件夹

三、将本地博客部署到Github

　　想要在互联网上浏览到我们的博客，就必须要将其同步部署到Github上，这样别人才能看到。

　　很多人可能会问Github网站没有中文版吗？的确没有中文版，如果你认为你的英文比较渣的话，那就只能推荐你使用中文脚本插件了。首先为浏览器安装油猴脚本管理器(Tampermonkey) https://tampermonkey.net/ ，然后安装GitHub汉化插件 https://github.com/52cik/github-hans ，安装完成后刷新页面，就是你需要的。

1. 创建Github账号和托管仓库

  • 注册Github账号

    地址: https://github.com/
    输入用户名、邮箱、密码，然后点击注册按钮。接下来是选择协议和服务套餐计划，选择免费服务套餐计划的那一项即可。
    用户名最好认真选取（虽然以后也可以修改），因为最后你的网站网址就是根据你的用户名来创建的，你不会想你的网站地址让人感觉不舒服吧。

  • 创建项目托管仓库

    登录Github后找到页面右上角处的加号‘+’，点击倒三角后选择‘New repository’，然后在‘Repository name’中填上“yourname.github.io”，到页面最下面点击创建。其中“yourname“是你注册时的用户名，和前面的‘Owner’是一样的，这个就是你网站的地址，是固定格式不允许更改的，例如用户名是woyouqian，那么网站地址就是woyouqian.github.io。

  • 生成SSH秘钥

    回到之前的命令窗口，如果关闭请右键重新打开，输入下面的命令，并将邮箱地址换成你自己的邮箱，然后连续按3下Enter键（跳过密码设置），如果你想设置密码请一下一下按，并根据提示输入两次密码，密码请牢记，以后同步本地博客到GitHub上时会要求密码验证。然后等待系统完成，SSH秘钥就生成了：

    如果你想输入密码请注意以下提示：
    Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<按Enter键就好>
    然后系统会要你输入密码：
    Enter passphrase (empty for no passphrase):<输入密码>
    Enter same passphrase again:<再次输入密码>

  • 在GitHub上添加SSH密钥

    到你的磁盘C:\Users\用户文件夹\ .ssh\ 位置处找到id_rsa和id_rsa.pub两个文件（注：id_rsa是私人密钥，id_rsa.pub是公共密钥）。使用记事本、word等软件打开id_rsa.pub并将其中内容完整复制出来。
    在浏览器上打开已登录的GitHub网页，找到并点击页面最右上角的倒三角，选择 ’Settings‘，在页面的左侧点击’SSH and GPG keys‘；或者直接打开链接https://github.com/settings/keys 。将你刚刚复制的秘钥粘贴到“key”中；“Title”无所谓随便填或不填，然后点击’Add SSH key‘即可。
